Section 1

What is HTML?

HTML stands for HyperText Markup Language. It is the standard language used to create and structure web pages. Think of it as the skeleton — the load-bearing structure that everything else is built on top of.

HTML is NOT a programming language. It cannot perform calculations, make decisions, or store data. It describes structure — it tells a browser what things are, not what to do with them.
Headings

Title and section titles that give content hierarchy and meaning.

Paragraphs

Blocks of text content that form the main body of a page.

Images

Visual content embedded into pages to communicate visually.

Hyperlinks

Connections between pages and resources — the "hyper" in HTML.

Tables

Organized grids of data — results, schedules, comparisons.

Forms

Input fields, buttons, dropdowns — the user's voice on a webpage.

HTML Powers Every Website You've Ever Visited
Google
YouTube
Amazon
GitHub

Every single one of these — the navigation, the buttons, the cards, the headings — is structured by HTML. Without it, browsers have nothing to render.

Real World Analogy — Think of a House

HTML builds the walls, floors, doors, and rooms — the entire structure.
CSS paints the walls, chooses the furniture, adds the aesthetic — everything visual.
JavaScript wires up the electricity, installs the smart locks, adds the interactivity.

You cannot decorate a house that doesn't exist. You cannot wire electricity through rooms that haven't been built. HTML comes first. Always.

Section 3

HTML, CSS, and JavaScript

Modern websites are built with three technologies that each have a distinct, irreplaceable role. Think of them as three members of one team — each doing what only they can do.

HTML
Structure — The Skeleton

HTML defines what content exists and what it means. A paragraph is a paragraph. A heading is a heading. An image is an image. HTML labels everything so browsers (and search engines, and screen readers) understand the page.

Headings Paragraphs Images Forms Tables
CSS
Presentation — The Appearance

CSS takes the raw structure HTML provides and makes it beautiful. Colors, fonts, spacing, layouts, animations, responsive behavior — all CSS. Without it, websites look like unformatted text documents.

Colors & Fonts Layouts Animations Responsive Design
JavaScript
Behavior — The Brain

JavaScript makes pages dynamic and interactive. It responds to user actions, updates content without reloading the page, validates forms, fetches data, and powers everything from dropdown menus to full-blown web applications.

Form Validation Dynamic Content API Calls Interactivity
The Car Analogy

HTML is the car's frame and body — the structure that makes it a car.
CSS is the paint, interior design, seat covers — the look and feel.
JavaScript is the engine, brakes, and infotainment — the things that actually run and respond.

You don't need to learn all three at once. This course masters HTML first. When your structure is solid, CSS and JavaScript become dramatically easier to learn.

Important Course Scope This course covers HTML only — no CSS, no JavaScript. That's not a limitation. That's the strategy. Students who try to learn all three simultaneously often struggle with all three. Master this one, and the others become much easier.
Section 4

How Websites Work

When you type a URL and press Enter, an entire sequence of events fires off in milliseconds. Here's exactly what happens — and why it matters for understanding what HTML actually does.

The Request-Response Cycle
User
Types a URL
Browser
Sends request
Server
Processes it
Files
Sent back
Webpage
Displayed!
Step-by-Step Breakdown
1
User Enters a URL

You type www.example.com into your browser's address bar and press Enter.

2
Browser Sends a Request

Your browser contacts a DNS server to find the IP address of that domain, then sends an HTTP request to the appropriate web server.

3
Server Processes the Request

The server receives the request, locates the correct files (HTML, CSS, JS, images), and prepares a response.

4
Server Sends Files Back

The server responds with the HTML file first, then the browser reads it and requests any additional CSS, JavaScript, and image files referenced inside.

5
Browser Renders the Page

The browser parses the HTML, applies the CSS, executes the JavaScript, and paints pixels on your screen. You see a webpage. All of this happens in under one second.

Why Does This Matter to You? You're learning HTML — the very first file the browser reads. When a page loads, the browser reads your HTML first and builds the entire structure from it. Understanding this flow helps you understand why HTML structure matters so much for performance, SEO, and accessibility.
Section 5

Web Browsers

A web browser is not just a window to the internet. It's a complex application that reads your HTML, applies CSS, runs JavaScript, and turns code into a visual experience — all in real time.

Popular Web Browsers
Google Chrome
Most popular globally. Recommended for this course.
Blink
Microsoft Edge
Chromium-based. Built into Windows 10/11.
Blink
Mozilla Firefox
Open source. Great developer tools.
Gecko
Safari
Apple's browser. Default on Mac & iPhone.
WebKit
What a Browser Actually Does
Reads and parses your HTML document from top to bottom
Builds a Document Object Model (DOM) — a tree of elements
Applies CSS styling to each element in the DOM
Executes JavaScript to add behavior and dynamic content
Paints the final visual result on your screen (pixel by pixel)
Handles user interactions — clicks, scrolls, keyboard input
Every browser has Developer Tools (press F12). This is where you'll inspect your HTML live. Get comfortable with it early — it's your best debugging tool.
Section 6

Setting Up VS Code

Visual Studio Code (VS Code) is a free, lightweight code editor by Microsoft. It's the industry standard for frontend developers — and the tool you'll use throughout this entire course.

Why VS Code?
Lightweight — Starts fast, runs lean. Doesn't slow down your computer.
IntelliSense — Auto-completes your HTML tags and attributes as you type.
Extensions — Add Live Server, Prettier, and dozens of tools in one click.
Industry Standard — The editor professional developers actually use every day.
100% Free — Open source. No subscriptions. No hidden costs.
Installation Steps
1
Go to code.visualstudio.com

Visit the official VS Code website. The download button auto-detects your operating system.

2
Download the Installer

Choose the correct version for Windows, Mac, or Linux and download the installer file.

3
Run and Install

Follow the installation wizard. The default settings are perfect for beginners.

4
Install Live Server Extension

Open VS Code → Extensions tab (Ctrl+Shift+X) → Search "Live Server" → Install. This extension auto-refreshes your browser when you save files.

Live Server — Your Best Friend Once Live Server is installed, right-click your HTML file in VS Code and select "Open with Live Server." Every time you save (Ctrl+S), your browser updates instantly. No manual refreshing needed throughout this entire course.
Section 7

Your First HTML File

This is the moment. Everything you've learned in this chapter has been leading here. Let's create your first webpage — and actually see it in a browser.

Step 1 — Create Your Project Folder
A
Create a folder on your Desktop (or Documents)

Name it: HTML-Mastery

B
Open that folder in VS Code

File → Open Folder → Select HTML-Mastery. This becomes your workspace for the course.

Step 2 — Create Your HTML File

In VS Code, right-click in the Explorer panel and select "New File." Name it:

index.html
Why index.html? Browsers look for "index.html" by default when they open a folder. It's the conventional name for the main page of any website.
Step 3 — Type This Code
HTML-Mastery / index.html
1<h1>Hello World</h1>
2
3<p>Welcome to HTML Mastery with Nilansha.</p>
4<p>My HTML journey begins today.</p>
Type it. Don't copy it. Your fingers need to build the memory that your eyes alone cannot. The act of physically typing HTML is not symbolic — it's how the learning actually gets stored.
Step 4 — Open in Browser
✓
Right-click index.html in VS Code → "Open with Live Server"

Your browser opens and you'll see "Hello World" as a large heading and two paragraphs below it.

Congratulations.

You just created your first HTML webpage. What you see in the browser is real HTML rendering. The browser read your two tags — <h1> and <p> — and turned them into a formatted webpage. This is HTML working. Remember this moment.

Watch Out

Common Beginner Mistakes

Thinking HTML is a Programming Language
"HTML is a programming language where you write code to make things happen."
HTML is a markup language. It describes structure and content. It does not perform calculations, loops, or logic.
Trying to Learn CSS Before Mastering HTML
Jumping into CSS or JavaScript before understanding how HTML structure works.
HTML comes first. Without a strong structural foundation, CSS and JS become confusing and frustrating.
Copying Examples Instead of Writing Them
Copy-pasting code examples to "save time" during practice sessions.
Type every example yourself. Copy-paste feels faster but produces zero learning. Your fingers need to learn the patterns, not just your eyes.
Ignoring Browser Developer Tools
Never pressing F12 or learning to inspect HTML in the browser.
Developer Tools (F12) are your best friend. You can see exactly how any website is built. Press F12 right now on any page and start exploring.
Interview Prep

Interview Questions

These questions appear in college exams, internship interviews, and developer job interviews. Try to answer each one before revealing the answer.

01
What does HTML stand for?
HTML stands for HyperText Markup Language. "HyperText" refers to text that contains links to other texts or resources. "Markup Language" means it uses tags to annotate and structure content.
02
Is HTML a programming language? Explain your answer.
No. HTML is a markup language, not a programming language. Programming languages (like Python or JavaScript) have logic, conditions, loops, and variables. HTML only describes the structure and content of a webpage — it cannot make decisions or perform calculations.
03
What is the role of HTML in web development?
HTML provides the structure and content of web pages. It defines what elements exist on a page — headings, paragraphs, images, forms, links — and gives them semantic meaning. It is the foundational layer that CSS and JavaScript are built on top of.
04
What is the difference between frontend and backend development?
Frontend is everything a user sees and interacts with — built with HTML, CSS, and JavaScript, running in the browser. Backend is the server-side logic, databases, and business rules that operate invisibly — built with languages like Python, Node.js, or Java, running on servers.
05
What is the purpose of CSS and JavaScript? How do they relate to HTML?
CSS (Cascading Style Sheets) controls the visual presentation — colors, fonts, layouts, and animations. JavaScript adds interactivity and dynamic behavior — form validation, menus, content updates. Both depend on HTML as the structural foundation they operate on. HTML defines what things are; CSS defines how they look; JavaScript defines what they do.
06
Explain what happens when a user visits a website.
1. The user types a URL in their browser. 2. The browser sends an HTTP request to the server via DNS resolution. 3. The server processes the request and locates the required files. 4. The server sends HTML, CSS, JavaScript, and media files back. 5. The browser parses HTML, builds the DOM, applies CSS, executes JavaScript, and renders the visual page on screen.
07
What is a web browser and what are its responsibilities?
A web browser is software that retrieves, interprets, and displays web content. Its responsibilities include: reading and parsing HTML, applying CSS styling, executing JavaScript, rendering the visual output, handling user interactions (clicks, scrolls), and managing navigation between pages.
08
What is a rendering engine? Name the engines for Chrome, Firefox, and Safari.
A rendering engine is the core component of a browser that parses HTML and CSS and draws the visual output. Chrome and Edge use Blink. Firefox uses Gecko. Safari uses WebKit. Different engines can sometimes render the same HTML slightly differently, which is why cross-browser testing is important.
09
Why is VS Code widely used by web developers?
VS Code is popular because it is free, open-source, lightweight, and highly extensible. It provides IntelliSense (smart autocompletion), syntax highlighting, integrated terminal, Git support, and thousands of extensions. It supports all major web technologies and is actively maintained by Microsoft with a large developer community.
10
What is the purpose of the Live Server extension in VS Code?
Live Server launches a local development server and opens your HTML file in a browser. Whenever you save your file (Ctrl+S), the browser automatically refreshes to show the latest changes. This eliminates manual browser refreshing and dramatically speeds up the development-feedback loop when learning HTML.
